# 시작합시다.

## 프로젝트 개요
- SSAFY 12반 학생들은 반 운영비로 간식을 구매해 나눠먹는다.
- 이 과자들에 대한 선호를 조사하고, 원하는 과자를 추천할 수 있는 플랫폼을 만들고자 한다.
- 필요한 다음은 다음과 같다.
  - 로그인/회원가입
  - 구매한 과자 관리 : CRUD
    - 좋아요/싫어요 기능
    - 댓글 기능
      - 좋아요/싫어요 기능 : CRUD
  - 과자 추천 관리 : CRUD
    - 좋아요/싫어요 기능
    - 댓글 기능 : CRUD
      - 좋아요/싫어요 기능
  - 관리자 기능 : 유저관리
    - 사용자 상태, 비밀번호 재설정 등을 사용 가능

- 사용자 role은 사용자, 관리자로 나눠진다.
  - 사용자 : 
    - 가능한 것
      - 과자 추천 게시글 작성
      - 과자 추천 게시글 좋아요/싫어요 기능, 댓글 기능
      - 구매한 과자 게시글 좋아요/싫어요 기능, 댓글 기능
      - 자신이 작성한 과자 추천 게시글 수정/삭제
      - 자신이 작성한 댓글 수정/삭제 기능
    - 불가능한 것
      - 구매한 과자 게시글 게시/수정/삭제
      - 타인이 작성한 과자 추천 게시글, 댓글 수정/삭제
  - 관리자 : 
    - 가능한 것
      - 일반 사용자가 할 수 있는 모든 기능
      - 자신이 작성한 과자 추천/구매한 과자 게시글 작성, 수정, 삭제
      - 타인이 작성한 과자 추천 게시글 삭제
      - 관리자 기능 (유저관리 기능)

- 기능 세부 명세
- 로그인/로그아웃
  - ID/PW를 사용해 로그인
  - PW는 Salt를 적용한 해시값과 salt를 저장
  - ID는 중복될 수 없지만, Unique한 ID를 추가로 관리해야함
  - 닉네임, 가입시간, 상태(활성화|비활성화|삭제됨), 작성한 게시글/댓글 숫자를 관리해야함
  - 회원가입/로그인 시 이메일 유효성 검증 필요
  - 비밀번호는 6자 이상 숫자,영문자 사용 가능
  - 비밀번호 특수문자는 일반적인 로그인 로직에서 허용하는 특수문자만을 허용 (SQL, 프로그래밍 코드 주입 방지 필요)
- 구매한 과자
  - 과자 이름은 중복될 수 있으며, Unique한 ID를 별도 관리해야함
  - 관리자가 구매한 과자 목록을 게시해 공유 가능
  - 사용자는 댓글과 좋아요/싫어요 기능으로 피드백 가능
  - 구매한 과자는 "배송중|재고있음|재고없음" 상태를 가짐
  - 상태 수정은 모든 사용자가 변경 가능
  - 페이지 당 5개까지 페이지네이션 적용
- 과자 추천
  - 과자 이름은 중복될 수 있으며, Unique한 ID를 별도 관리해야함
  - 사용자가 새로 주문할 과자를 추천할 수 있음
  - 과자 이름과 추천하는 이유를 적을 수 있음
  - 댓글과 좋아요/싫어요로 피드백이 가능함
  - 페이지 당 5개까지 페이지네이션 적용
- 좋아요/싫어요
  - 게시글과 댓글에 대한 피드백 기능
  - 한 사람 당 좋아요 또는 싫어요 하나의 피드백만 가능 (토글)
  - 좋아요 또는 싫어요에 1초 이상 마우스 호버 시 피드백을 남긴 사용자의 닉네임을 확인 가능
- 댓글
  - 사용자는 게시글에 댓글을 달 수 있음
  - 대댓글 가능
  - 댓글에 대해서도 좋아요/싫어요 댓글을 사용할 수 있음
- 마이페이지
  - 지금까지 작성한 댓글/게시글을 한눈에 볼 수 있음
  - 닉네임과 비밀번호를 수정할 수 있음
  - 회원 탈퇴가 가능
- 관리자 페이지
  - 회원 정보를 조회할 수 있으며 (ID/닉네임/게시글, 댓글 수/상태) 상태와 비밀번호를 변경 가능

- 아키텍쳐
  - Next.js와 Spring, MySQL을 사용
  - Next.js
    - MVC 중 View,Controller를 담당
    - 백엔드의 API 호출이 필요한 경우, 반드시 서버 컴포넌트에서 호출해 백엔드 주소를 노출시키지 않아야함
    - 페이지를 구성하는 구성 요소를 컴포넌트화 하여 유지보수성을 확보해야함
    - Zustand와 Axois를 활용해 상태관리와 API 관리가 필요함
      - Zustand는 상태관리를 담당
      - Axios는 API 관리와 호출 코드 모듈화 담당
        - 별도 파일로 백엔드 서버 주소와 호출 가능한 API 목록을 관리할 수 있음
        - 최근 발생한 공급망 공격이 일어난 버전을 지양
    - Vitest를 활용해 Unit Test 코드 작성 필요
      - 기능에 테스트 코드를 맞추면 안됨
      - 구현 코드가 바뀌어도 작동 가능한 유지보수성 있는 테스트코드 작성
      - - 적절한 수준으로 나뉜 로그 시스템 구현 필요 (INFO, ERROR, DEBUG)
      - 로그는 날짜로 구분되는 파일로 저장되며, 1주일이 지난 로그는 자동 삭제
  - Spring
    - MVC 중 Model을 담당
    - DTO/DAO 디자인을 사용해 MySQL과 Next.js를 조율
    - 인증/인가를 담당 (Spring Security)
      - ID/PW 기반의 인증과 JWT+refresh 토큰을 활용한 인가 기능 필요
    - Controller, Config, Service, Dto, Entity, Repository 등등으로 모듈화하여 유지보수성을 확보해야함
    - JPA를 활용하여 자바 코드를 이용해 엔티티 관리
    - 설정파일에서 프론트엔드 주소, 데이터베이스 주소와 계정 정보, JWT토큰 키 등을 중앙화된 관리가 가능해야함
    - 요청에 대한 권한, 입력값 유효성 검증이 필요함
  - JTest를 활용한 Unit 테스트 코드 작성필요
    - 기능에 테스트 코드를 맞추면 안됨
    - 구현 코드가 바뀌어도 작동 가능한 유지보수성 있는 테스트코드 작성
  - 적절한 수준으로 나뉜 로그 시스템 구현 필요 (INFO, ERROR, DEBUG)
    - 로그는 날짜로 구분되는 파일로 저장되며, 1주일이 지난 로그는 자동 삭제
